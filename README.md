# Задание 1. Проектирование объектной модели ML-сервиса

## Цель работы

Спроектировать объектную модель ML-сервиса на основе требований общего задания курса.

Проектирование в виде Python-кода.

---

## Выделенные сущности

**В модели выделены сущности:**

- пользователь `User`
- администратор `Admin`
- баланс пользователя `UserBalance`
- ML-модель `Model`
- простая модель `SimpleModel`
- задача `Task`
- результат задачи `TaskResult`
- операция с балансом `MoneyMove`
- пополнение `PlusMoney`
- списание `MinusMoney`
- ошибка в данных `DataError`

---

**При проектировании учитывались требования задания:**

- есть пользователь;
- есть ML-модель;
- есть ML-задача;
- есть отдельная сущность баланса пользователя;
- есть история транзакций;
- есть результат предсказания;
- есть проверка положительного баланса;
- есть обработка невалидных строк;
- данные авторизации пользователя хранятся в виде хэша пароля.

---

## Логика связей между сущностями

### Пользователь

Сущность `User` хранит:

- идентификатор пользователя;
- email;
- хэш пароля;
- роль пользователя.

Пользователь **не управляет балансом напрямую**.

### Баланс пользователя

Сущность `UserBalance` вынесена отдельно и связана с пользователем по `user_id`.

Она хранит:

- `user_id`;
- текущий баланс;
- историю операций.

Через `UserBalance` выполняются:

- пополнение баланса;
- списание средств;
- проверка достаточности средств;
- просмотр истории операций.

### Транзакции

Базовый класс `MoneyMove` содержит общие поля для движения средств:

- id операции;
- `user_id`;
- сумма;
- дата и время;
- `task_id`, если операция связана с ML-задачей.

Наследники:

- `PlusMoney` — пополнение;
- `MinusMoney` — списание.

### ML-модель

Сущность `Model` хранит:

- id модели;
- название;
- описание;
- стоимость предсказания;
- признак активности.

Наследник `SimpleModel` реализует простой метод `predict()`.

### ML-задача

Сущность `Task` хранит:

- id задачи;
- ссылку на пользователя;
- ссылку на баланс пользователя;
- ссылку на модель;
- входные данные;
- статус выполнения;
- ошибки валидации;
- результат работы.

### Результат

Сущность `TaskResult` хранит:

- id результата;
- id задачи;
- ответы модели;
- число валидных строк;
- число ошибочных строк;
- стоимость выполнения.

---

## Применение принципов ООП

### Инкапсуляция

Использованы скрытые поля:

- `_password_hash`
- `_amount`
- `_moves`
- `_status`

Баланс изменяется не напрямую, а через методы сущности `UserBalance`.

### Наследование

Показано через:

- `Admin` наследуется от `User`
- `SimpleModel` наследуется от `Model`
- `PlusMoney` и `MinusMoney` наследуются от `MoneyMove`

### Полиморфизм

Показан через переопределение методов:

- `do()` в `PlusMoney` и `MinusMoney`
- `predict()` в `SimpleModel`
