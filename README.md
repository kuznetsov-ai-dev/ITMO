## Задание 1

### Начало работы

Цель работы — спроектировать объектную модель ML-сервиса.

Проектирование выполнено в виде кода.

**В модели выделены сущности:**

- пользователь `User`
- администратор `Admin`
- ML-модель `Model`
- простая модель `SimpleModel`
- задача `Task`
- результат задачи `TaskResult`
- операции с балансом `MoneyMove`
- пополнение `PlusMoney`
- списание `MinusMoney`
- ошибки в данных `DataError`

**При построении модели я учитывал требования задания:**

- есть пользователь
- есть ML-модель
- есть ML-задача
- есть история транзакций
- есть результат
- есть проверка баланса
- есть обработка невалидных строк


### Принципы 

**Инкапсуляция**

показана через скрытые поля:

- `_balance`
- `_password`
- `_moves`
- `_status`

Баланс меняется не напрямую, а через методы пополнения и списания.

**Наследование**

показано так:

- `Admin` наследуется от `User`
- `SimpleModel` наследуется от `Model`
- `PlusMoney` и `MinusMoney` наследуются от `MoneyMove`

**Полиморфизм**

через переопределение методов:

- `do()` в классах `PlusMoney` и `MinusMoney`
- `predict()` в классе `SimpleModel`
