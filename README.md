# Задание 6. Web-интерфейс личного кабинета ML-сервиса

## Описание

В проекте реализован Web-интерфейс личного кабинета ML-сервиса поверх существующего backend-ядра.

Система поддерживает:

- регистрацию и авторизацию пользователей;
- личный кабинет;
- отображение и пополнение баланса;
- выполнение ML-запросов;
- проверку баланса перед выполнением запроса;
- частичную валидацию данных;
- отображение результатов и истории операций;
- REST API для тех же сценариев;
- дополнительную роль администратора.

---

## Архитектура

Проект состоит из следующих частей:

- `FastAPI app` — backend API и Web-интерфейс;
- `PostgreSQL` — хранение пользователей, баланса, транзакций и истории предсказаний;
- `RabbitMQ` — очередь задач;
- `worker-1`, `worker-2` — асинхронная обработка ML-задач;
- `nginx` — reverse proxy.

Web-интерфейс реализован через Jinja templates и cookie/JWT-аутентификацию.

REST API продолжает работать отдельно через HTTP Basic Auth.

---

## Реализованный функционал

### 1. Главная страница

Главная страница доступна без авторизации и содержит:

- описание сервиса;
- перечень возможностей;
- список доступных моделей;
- демо-учётные записи.

### 2. Регистрация и авторизация

Поддерживается:

- регистрация нового пользователя;
- вход по логину или email;
- cookie/JWT-аутентификация для web;
- HTTP Basic Auth для REST API.

### 3. Баланс пользователя

Пользователь может:

- посмотреть текущий баланс в личном кабинете;
- пополнить баланс вручную;
- увидеть изменения в истории транзакций.

### 4. Выполнение ML-запросов

Поддерживаются три варианта ввода данных:

- одна запись в JSON;
- список записей в JSON;
- CSV-файл.

После отправки запроса:

- задача ставится в RabbitMQ;
- воркер обрабатывает данные;
- при успехе пользователь видит результат в истории;
- кредиты списываются только после успешной обработки.

### 5. Валидация данных

Если часть строк некорректна:

- некорректные строки попадают в `rejected_rows`;
- корректные строки попадают в `accepted_rows`;
- пользователь видит, какие данные обработаны, а какие отклонены.

### 6. Проверка баланса

Перед запуском ML-задачи проверяется:

- баланс больше нуля;
- средств достаточно для стоимости модели.

При недостатке средств задача не создаётся.

### 7. История

В личном кабинете доступна история:

- ML-запросов;
- загруженных данных;
- результатов предсказаний;
- списаний и пополнений.

### 8. Администратор

Дополнительно реализована роль администратора.

Администратор может:

- смотреть всех пользователей;
- пополнять баланс любому пользователю;
- смотреть все транзакции в системе.

## REST API примеры

### Регистрация

~~~bash
curl -X POST http://localhost/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "login": "new_user",
    "email": "new_user@mail.com",
    "password": "123456"
  }'
~~~

### Авторизация

~~~bash
curl -X POST http://localhost/auth/login \
  -u new_user:123456
~~~

### Пополнение баланса

~~~bash
curl -X POST http://localhost/balance/deposit \
  -u new_user:123456 \
  -H "Content-Type: application/json" \
  -d '{
    "amount": "100.00",
    "description": "manual deposit"
  }'
~~~

### ML-запрос

~~~bash
curl -X POST http://localhost/predict \
  -u new_user:123456 \
  -H "Content-Type: application/json" \
  -d '{
    "model": "demo_model",
    "rows": [
      {
        "row_id": "good-1",
        "features": {"x1": 1.2, "x2": 5.7}
      },
      {
        "row_id": "bad-1",
        "features": {"x1": "oops", "x2": 10}
      }
    ]
  }'
~~~

### Проверка статуса

~~~bash
curl http://localhost/predict/TASK_ID \
  -u new_user:123456
~~~

### История предсказаний

~~~bash
curl http://localhost/history/predictions \
  -u new_user:123456
~~~

### История транзакций

~~~bash
curl http://localhost/history/transactions \
  -u new_user:123456
~~~
